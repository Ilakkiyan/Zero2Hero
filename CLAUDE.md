## What this is

**Zero2Hero — the honest AI cofounder for ambitious students.** It turns a vague
idea into an evidence-backed go/no-go decision: interview → de-risked plan →
adversarial challenge → cheapest **real-world** test → evidence-aware confidence →
honest **Build / Don't / Not yet** verdict → first version + launch kit.

Built for the **USAII Global AI Hackathon 2026** (College track — "AI for Life &
Work"; it's a student-only competition, and there's a **Responsible AI Award** we
care about). Submission deadline: **2026-06-21**.

## Audience & positioning (committed)

- **User = an ambitious student** with an early-stage idea (class project, hackathon
  build, or startup) who's about to sink real time into it without knowing if it's
  worth it. Earlier drafts said "solo first-time founder"; we recentered on
  **students** because the hackathon is student-only and judging rewards how well
  you understand *the user* (a student team credibly understands student users).
- **Core differentiator = honesty.** Most AI hypes the user's idea; Zero2Hero
  pressure-tests it and is willing to say "don't build this." This anti-sycophancy
  stance is also our Responsible-AI story.
- **Tagline style:** the user dislikes em dashes in submission/marketing copy — use
  commas or periods there. (The existing in-app/README copy still uses em dashes;
  only change copy when asked.)

## Invariants / product mechanics (don't break these)

- **Honesty is architectural.** The Verdict only says "Build it" when a high-risk
  assumption *passed* AND is backed by supporting **field** (real-world) evidence —
  it cannot be talked into a yes ([lib/verdict.ts](lib/verdict.ts)). A failed
  high-risk assumption forces a kill/pivot.
- **Never assume software.** Field tests, first version, and launch channels adapt
  to the *kind* of idea — offline/manual (flyers, sign-up sheets, pre-orders,
  concierge, local channels) is first-class, not a fallback. The prompts enforce
  this; keep it that way.
- **Structured output is validated.** All model JSON is zod-checked at the API
  boundary (422 on mismatch). This is what makes it a system, not a chat wrapper.
- **Field-test capture is calibrated against pre-registered thresholds.** It judges
  a result against the design's own proveIf/killIf (pass only if the bar is met,
  fail only if the kill bar is hit, else inconclusive). Calibration is
  **model-sensitive** — a worked example in the prompt was needed to stop gpt-4o-mini
  rounding a near-miss up to "passed." Re-verify if you change models.
- **Human-in-the-loop.** The AI proposes; the user applies every change. Nothing
  auto-commits to the plan.
- **Hallucination control.** Research drops citations with unknown assumption ids,
  invalid stances, or non-real URLs server-side.
- **Safety guardrails.** Clearly harmful/illegal ideas are refused server-side
  before any model call (interview streams a refusal; plan returns 422), plus a
  prompt-level refusal. The screen is conservative to avoid false positives
  (e.g. "Nerf gun marketplace" and "malware-detection tool" must pass).

## Providers & running it

- **Local-first by default:** Ollama + Qwen2.5 (`qwen2.5:14b` default for reliable
  judgment; `qwen2.5:7b` lighter/faster — pick in Settings). `npm run llm:pull` /
  `llm:pull:7b`.
- **Cloud option:** Azure OpenAI (`gpt-4o-mini`). Per-request switch via the
  header toggle / `x-llm-provider`. Server default = `LLM_PROVIDER` (ollama).
- **Secrets live only in `.env.local`** (git-ignored; copy from `.env.example`).
  Never commit keys. For Azure set `AZURE_OPENAI_*`. Web research uses a local
  SearxNG (Docker): `docker compose -f docker-compose.searxng.yml up -d`.
- Run: `npm install && npm run dev` (http://localhost:3000). Desktop:
  `npm run desktop:dev`. Installers: `npm run dist` (output in `release/`).

### WSL notes
- Ollama: install in WSL or point `OLLAMA_BASE_URL` at a host Ollama instance.
- Electron desktop/`dist` needs a Linux GUI/deps; the web app (`npm run dev`) is
  the simplest path on WSL. SearxNG needs Docker available in WSL.

## Release

- Version **1.0.0**. `.github/workflows/release.yml` builds unsigned installers
  (.dmg/.exe/.AppImage) for mac/win/linux on a `v*` tag and attaches them to a
  GitHub Release. `v1.0.0` currently points at the latest commit (with guardrails).
- If you change code that should ship in installers, **re-cut the tag** (move
  `v1.0.0` to HEAD, or bump to `v1.0.x` and update `package.json` version + lockfile)
  so CI rebuilds.
